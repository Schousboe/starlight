Den hurtigste måde at oprette et nyt Starlight projekt, er at bruge `create astro` som vist i 
[kom-i-gang guiden](/da/getting-started/#opret-et-nyt-projekt).
Hvis du vil tilføje Starlight til et eksisterende Astro projekt, vil den her guide vise dig hvordan.

## Sæt Starlight op

For at følge denne guide har du brug for et eksisterende Astro projekt

### Tilføj Starlight integrationen

Starlight er en [Astro integration](https://docs.astro.build/en/guides/integrations-guide/). Tilføj den til dit projekt ved at køre `astro add` kommandoen i dit projekts root mappe:

<Tabs syncKey="pkg">
    <TabItem label="npm">
        ```sh
        npx astro add starlight
        ```

    </TabItem>
    <TabItem label="pnpm">
        ```sh
        pnpm astro add starlight
        ```
    </TabItem>
    <TabItem label="Yarn">
        ```sh
        yarn astro add starlight
        ```
    </TabItem>

</Tabs>

Det vil installere de nødvendige dependencies og tilføje Starlight til `integration` arrayen i dine Astro konfigureringsfiler.

### Konfigurer integrationen

Starlight integrationen er nu konfigureret i din `astro.config.mjs` fil.

Tilføj en `title` for at komme i gang:

```js ins={8}
// astro.config.mjs
import { defineConfig } from 'astro/config';
import starlight from '@astrojs/starlight';

export default defineConfig({
	integrations: [
		starlight({
			title: 'Min smukke docs side',
		}),
	],
});
```

Find alle tilgængelige muligheder i [Starlight konfigurations referencen](/reference/configuration/).

### Konfigurer content collections

Starlight er bygget ovenpå Astro's [content collections](https://docs.astro.build/en/guides/content-collections/), som er konfigurerede i `src/content.config.ts` filen.

Tilføj eller opdater content config filen, med at tilføje en `docs` kollektion der bruger Starlights [`docsLoader`](/reference/configuration/#docsloader) og [`docsSchema`](/reference/configuration/#docsschema):

```js ins={3-4,7}
// src/content.config.ts
import { defineCollection } from 'astro:content';
import { docsLoader } from '@astrojs/starlight/loaders';
import { docsSchema } from '@astrojs/starlight/schema';

export const collections = {
	docs: defineCollection({ loader: docsLoader(), schema: docsSchema() }),
};
```

Starlight supporter også [`legacy.collections`](https://docs.astro.build/en/reference/legacy-flags/) flaget, hvor collections bliver håndteret ved hjælp af legacy content collections implementationen.
Det er brugbart hvis du har et eksisterende Astro projekt og ikke kan lave nogle ændringer til collections til denne tid, for at bruge en loader.

### Tilføj content

Starlight er nu konfigureret og det er tid til at tilføje noget content!

Opret en `src/content/docs` mappe og start med at tilføje en `index.md` fil.
Dette vil være hjem til din nye side:

```md
---
# src/content/docs/index.md
title: Mine docs
description: Lær mere omkring mine projekter på denne docs side bygget med Starlight
---

Velkommen til mit projekt!
```

Starlight bruger fil-baseret routing, hvilket betyder enhver Markdown, MDX eller Markdoc fil i `src/content/docs/` vil blive lavet til en side på dit projekt. Frontmatter metadata (`title` og `description` felterne i eksemplet over) kan ændre hvordan hver enkelt side er fremstillet.
Se alle tilgængelige muligheder i [frontmatter referencen](/reference/frontmatter/).

## Tips for eksisterende projekter

Hvis du har et eksisterende Astro projekt, kan du bruge Starlight til hurtigt at tilføje en dokumentation sektion til dit projekt.

### Brug Starlight ad en subpath

For at tilføje alle Starlight sider til en subpath, placer alt dit docs indhold inde i en undermappe af `src/content/docs`.

For eksempel, hvis alle dine Starlight sider skulle starte med `/guides/`, tilføj dit indhold i `src/content/docs/guides` mappen:

import { FileTree } from '@astrojs/starlight/components';

<FileTree>

- src/
  - content/
    - docs/
      - **guides/**
        - guide.md
        - index.md
  - pages/
- astro.config.mjs

</FileTree>

I fremtiden, planlægger vi at supporte den her use-case bedre for at undgå behovet for den ekstra nestede mappe i `src/content/docs/`.

### Brug Starlight med SSR

For at aktivere SSR, følg ["On-demand Rendering Adapters"](https://docs.astro.build/en/guides/on-demand-rendering/) guiden i Astros docs for at tilføje en server adapter til dit Starlight projekt.

Dokumentations sider genereret af Starlight er pre-renderet automatisk, uanset dit projekts output metode. For at fravælge at dine Starlight sider pre-renderer, sæt [`prerender` konfigurations valget](/reference/configuration/#prerender) til `false`.
